# asset-agent — 交互设计（融合版）

文档状态：v1.0 草案
角色：产品经理
日期：2026-05-16

---

## 1. 产品定位

三位智能体（设计、前端、后端）在资产仓库中协作，管理者需要在一个页面内看到谁在做什么、做到哪了、花得值不值。

## 2. 设计原则

- **三合一**：结构、演化、审计三个视图放在同一个页面，不跳转
- **自上而下钻取**：选项目 → 看演化 → 查审计，信息粒度逐层细化
- **减法优先**：不加第四个视图，不加详情面板，不加事件时间线
- **零配置**：双击打开，file:// 协议可用

### 3.1 左侧资产树

用途：导航 + 职责可见。

- 展开/折叠目录，缩进表示层级
- 目录右侧显示 🤖 badge（该目录挂载的智能体）
- 点击目录选中，主面板切换到此项目的相关视图
- 搜索框过滤目录名和文件名

不包含：文件节点（目录层足够），智能体不占树节点（以 badge 形式出现）。

### 3.2 主面板 — 四个标签

#### 标签 1：结构

用途：展示选中目录下的资产详情。

- 目录下的文件列表（不递归，只看当前层）
- 每行显示：图标 + 文件名 + 所属智能体
- 智能体列显示该文件由哪个智能体产出

与左侧资产树的区别：树看全局分工，结构标签看局部资产组成。

#### 标签 2：演化

用途：展示选中项目各智能体的版本迭代。

- 三列（设计、前端、后端），从上到下排列版本号
- 蓝色实线箭头：trigger（设计驱动开发）
- 橙色虚线箭头：feedback（实现反馈设计）
- 底部标记收敛状态：全列无活跃 feedback 时显示 ✓ 已收敛
- 列高反映迭代轮数

数据来源：app.py 生成的版本追踪数据（qtcloud-asset 的演化图数据模型）。

#### 标签 3：审计

用途：展示选中项目最近的智能体调用记录。

- 时间倒序列表，每行一条调用记录
- 每行显示：时间 + 智能体 + token 数 + wasteRate + 边界状态
- wasteRate > 50% 的行标记 ⚠️
- 边界违规的行标记 🔴
- 点击行展开详情：文件加载清单（文件名、token、引用状态）

数据来源：agent 调用日志（qtcloud-agent 的审计数据模型）。

#### 标签 4：配置

用途：查看和编辑智能体的文件读取边界。

- 选中一个智能体，显示其 watch path 规则
- 显示匹配的文件列表和数量
- 编辑规则后模拟保存（当前为前端模拟，无后端）

### 3.3 交互流

典型用户操作路径：

```
1. 打开页面 → 左侧看到资产树，知道谁管什么
2. 点击"平台"目录 → 切换到结构标签，看到平台目录下的文件
3. 切换到演化标签 → 看到平台功能的三列版本迭代图
4. 发现某版本 feedback 密集 → 切换到审计标签
5. 审计列表按时间倒序显示该项目的调用记录
6. 找到对应时间段的调用 → 展开看具体加载了哪些文件、哪些浪费了
```

## 4. 数据流

```
                ┌────────────────┐
                │   事件数据源    │
                └───────┬────────┘
                        │
          ┌─────────────┼─────────────┐
          ▼             ▼             ▼
   ┌──────────┐ ┌──────────┐ ┌──────────┐
   │ 结构数据  │ │ 演化数据  │ │ 审计数据  │
   │ (目录树)  │ │(版本节点) │ │(调用日志) │
   └────┬─────┘ └────┬─────┘ └────┬─────┘
        │            │            │
        ▼            ▼            ▼
   ┌─────────────────────────────────────┐
   │        统一渲染层（单页三标签）      │
   │  ┌──────┐ ┌──────┐ ┌──────┐ ┌────┐ │
   │  │结构  │ │演化  │ │审计  │ │配置│ │
   │  └──────┘ └──────┘ └──────┘ └────┘ │
   └─────────────────────────────────────┘
```

三种数据共用同一份 data.js 全局变量，各自按标签页过滤。

## 5. 与现有原型的关系

| 组件 | 来源 | 融合方式 |
|------|------|---------|
| 资产树渲染 | qtcloud-asset index.html | 移植到左侧面板 |
| 版本演化图 | qtcloud-asset evolution.html | 移植到演化标签 |
| 审计列表 | qtcloud-agent audit.html | 移植到审计标签 |
| 边界配置 | qtcloud-agent plan.html | 移到配置标签 |
| 数据格式 | 两套 data.js 共存 | 合并为统一 graphData |

保留两套 data.js 各自的生成逻辑（app.py 生成资产版 data.js，调用日志生成审计版 data.js），在渲染层通过事件筛选合并。

## 6. 技术实现

- 前端：vanilla HTML + vis-network（演化标签）
- 布局：flexbox，左侧 280px 固定，右侧自适应
- 数据：data.js 全局变量，按类型筛选
- 交互：点击标签切换，点击树节点过滤，点击审计行展开
- 零构建，双击即用
