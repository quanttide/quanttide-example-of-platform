# code-agent — 探索性任务的自定义智能体需求

## 背景

当前 Opencode 的默认行为假设指令是清晰的、验收标准是预先明确的。但探索性任务的特征相反：指令短、隐含假设多、验收标准在执行过程中才逐渐清晰。

## 需求

### R1：执行前理解确认

智能体在接到指令后，先输出对任务的理解（做什么、产出形态、验收标准），等待确认后再动手。

这不是建议性的——不理解不能继续。

### R2：执行中可中途纠正

智能体在执行长任务时，在关键节点（如写文件前、切换方向前）主动暂停并摘要当前状态，等待用户指示继续或调整。

### R3：交付前自检

智能体在交付产出前，对照 R1 中确认的理解自行检查：产出是否匹配约定的形态和标准。不匹配时主动说明差异。

### R4：方向变更时保留上下文

当用户说"不是这个方向"时，智能体能基于已完成的上下文重新理解意图，而不是从头开始。

### R5：过程可审计

用户能回溯智能体在执行过程中的关键决策点：当时理解了什么、为什么选择这个方案、中途调整了什么。

## 不覆盖的范围

- 不需要记忆跨会话的上下文（每次会话独立）
- 不需要自动执行重复任务（这是另一个 agent 的职责）
- 不需要与其他工具集成（MCP、外部 API）
