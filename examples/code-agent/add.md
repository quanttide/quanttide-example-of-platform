# code-agent — 意图对齐约束设计

## 现有可用的约束机制

| 机制 | 约束力 | 说明 |
|------|--------|------|
| AGENTS.md | 中 | AI 读取执行，但违反没有自动阻断 |
| Skills | 高 | 步骤不可跳过，跳了工作流断掉 |
| opencode.json | 高 | 可配置规则、权限、agent 行为 |
| 自定义 agent | 高 | 可限定工具集和行为模式 |

## 可选方案

### A：Skill 约束

定义一个执行 Skill，强制步骤顺序：

1. 接到指令后先输出理解确认（做什么、产出格式、验收标准）
2. 确认后执行
3. 交付前输出验收清单

跳步 = Skill 流程不完整，不可继续。

### B：AGENTS.md 硬规则

在 AGENTS.md 写入不可协商的执行前确认规则，AI 每次读入后按规则执行。

### C：opencode.json agent 配置

将 code-agent 配置为独立 agent，通过 opencode.json 限定行为。

## 各方案比较

| 维度 | A（Skill） | B（AGENTS.md） | C（opencode.json） |
|------|-----------|---------------|-------------------|
| 约束力 | 高 | 中 | 高 |
| 实现成本 | 中（写 Skill） | 低（加几行） | 中（配 agent） |
| 灵活性 | 结构化 | 自由 | 限定 |
