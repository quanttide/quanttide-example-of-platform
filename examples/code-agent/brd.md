# code-agent — 意图与验收对齐

## 问题

Opencode 经常误解我的意图、上下文和验收标准。

典型表现：

1. **交付形态不符** — 我说"整合"，我要的是代码，不是文档。我说"重写"，我要的是不同方案，不是同方案美化。
2. **验收标准在执行过程中才清晰** — 我自己也是在看到产出之后才想清楚"这不是我要的"。
3. **短指令 vs 复杂期望** — 一条短指令背后有几层隐含假设，没有说出来。

## 根因

不是 AI 不听话，是验收标准在我脑子里，不在任何文档里。我自己也是在看到产出后才明确"这不是我要的"。

对于探索性任务（方向会变，验收在执行中逐渐清晰），AGENTS.md 和 Skills 能控制偏差范围，但解决不了"我说一个词，你默认做完的标准和我不同"的问题。

## 方向

利用 Opencode 现有机制，不另起炉灶。可行的改进：

### 机制 1：指令附带验收信号

发出指令时同时说明"什么算做完"，而不是只说要做什么。例如：

- "写一个融合版 HTML" → "写一个融合版 HTML，内嵌数据双击即用，左侧导航右侧四标签"
- "重写" → "重写成首页设计"

### 机制 2：先对齐再动手

复杂任务先出方案再执行。发出指令后，AI 先输出"我理解你要做的是 X，产出是 Y 格式，验收标准是 Z"，等确认后再动手。

这不需要新工具——现有对话机制已经支持，只是缺少习惯。

### 机制 3：过程检查点

长任务（如本次会话的多步操作）在关键步骤后主动确认方向再继续，而不是一口气做完所有步骤。

## 不做的方案

- 不另建验收管理系统（Jira、TestCase 等）
- 不给 Opencode 加外部工具链
- 不写复杂的 YAML 规则文件描述验收标准

核心思路：**用对话机制解决对齐问题，而不是用工具解决。** 需要的不是新系统，是发出指令时多说一句话的习惯。
