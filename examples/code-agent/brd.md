# code-agent — 意图与验收对齐

## 问题

Opencode 经常误解我的意图、上下文和验收标准。

典型表现：

1. **交付形态不符** — 我说"整合"要的是代码，不是文档。我说"重写"要的是不同方案，不是同方案美化。
2. **验收在执行过程中才清晰** — 自己看到产出后才想清楚"这不是我要的"。
3. **短指令 vs 复杂期望** — 一条指令背后有几层隐含假设，没有说出来。

## 原因

对话约束没有执行力。我要求了它还是按自己默认的理解走。习惯不解决结构性矛盾。

## 需求

需要刚性约束——不靠对话习惯，靠机制强制对齐。用 Opencode 现有配置层实现，不另起炉灶。
