当前对话 · 意图模型

目标
用 Flutter 客户端 + 单个 Open Code 服务端，落地“左对话、右意图文档”的意图治理交互形态。客户端的核心定位：不可见的行为编排层，通过静默改变 AI 的输入与上下文，约束和引导 AI 的默认交互行为。

当前探索
客户端核心机制已明确，不引入代理服务端，所有编排逻辑留在 Flutter 层。已识别三个核心编排机制：

· 人类编辑同步：用户在右栏直接编辑意图文档后，客户端写文件，并隐式将新文档内容注入 AI 上下文
· 偏差修正：AI 维护文档出现偏差时，客户端可在下一轮对话前静默修正上下文
· 方向约束：对话漂移时，客户端通过调整系统提示中的约束权重来引导行为

双模式编辑方案已确定：AI 自动维护为默认路径，人类直接编辑为按需路径，编辑后自动同步上下文，不产生对话噪音。

约束

· Flutter 客户端自研，所有编排逻辑在客户端完成
· Open Code 服务端直接复用，不做任何修改，不引入代理服务端
· 单服务端实例，AI 同时承担对话和意图文档维护
· 左对话、右意图文档的交互布局已定
· 意图文档以文件形式存在于 Open Code 工作区，AI 直接编辑
· 客户端监听文件变化并渲染右栏
· 编排行为对 AI 透明，对用户可感知结果但不可见编排过程

状态
客户端设计哲学已确立，核心机制和交互方案已定。
下一步：梳理客户端编排层的完整行为清单，设计状态机，明确各行为的触发条件和优先级，开始编码。
