项目管理原则

三级资产模型

层级 稳定度 职责 生命周期
示例 可丢弃 验证单个模式，回答一个具体问题 验证完成即归档或删除
库 稳定 骨架性的数据流转规则，不依赖UI和业务 长期维护，审慎修改
平台 中等 胶水代码，组合库完成交互和业务 随交互迭代持续重写

提炼路径

· 示例 → 库：识别到独立的数据流转骨架（如状态机），直接提炼
· 示例 → 平台：非骨架的胶水逻辑，验证通过后搬入平台
· 平台 → 库：平台代码中浮现出未预见但确实有效的骨架，回头提炼

示例管理

· 独立开，不聚合。一个示例验证一个模式
· 不依赖前一个示例的代码，依赖的是已提炼库的接口
· 提炼完成后原示例归档或删除，不留旧代码负担

早期治理

· 君主制：创始人直接定骨架（脚手架宪法），快速启动
· 后门开放：任何早期决策都可被修正，不需要走流程
· 未来迁移：平台代码浮现真宪法时，君主制自然过渡到议会制
