意图澄清工具 — 交互设计说明（v2）

1. 核心概念

工具围绕意图模型构建。意图模型是AI对“人类当前想做什么、怎么想、边界在哪、进展如何”的结构化理解，包含四个维度：

· 目标：要达成的最终状态或核心问题
· 当前探索：正在尝试的方向、方法或假设
· 约束：显式或隐式的边界条件
· 状态：当前阶段及已确认/否定的结论

意图模型直接作为界面展示给人类，而非拆分为卡片。它是人与AI之间的唯一对齐基准，既是高质量上下文，也可导出为BRD。

2. 桌面端布局

双栏等分，无装饰元素。

```
│ │ [意图模型已更新:       │ │ └──────────────────┘  │ │
│ │  探索→插件架构验证中]  │ │                      │ │

│ │ 输入框                │ │  约束                 │ │
│ └──────────────────────┘ │ ┌──────────────────┐  │ │

│                          │ └──────────────────┘  │ │
│                          │                      │ │
│                          │  状态                 │ │

```

左栏：对话区

· 标准聊天界面，支持自然语言输入。
· AI在对话中检测到意图变化时，会附加一条系统提示：“意图模型已更新：[变化摘要]”，但不打断对话流。

右栏：意图模型面板

· 四个字段（目标/当前探索/约束/状态）以可编辑文本块展示，每个字段都是一个轻量文本编辑器，点击即可直接修改。
· 人类可随时手动编辑任意字段，修改后AI下一次请求自动携带最新意图模型。
· 面板底部显示最后更新时间，以及“导出 BRD”按钮（将意图模型拼接为Markdown文档）。

单栏对话流，顶部常驻可折叠的意图模型区域。

· 顶部意图区域默认折叠（只显示“当前意图 (展开▼)”），点击展开后显示四个可编辑字段。
· 字段编辑方式同桌面端，直接点击修改。
· 对话流中AI提示意图更新时，顶部区域自动同步刷新。
· 导出按钮置于输入框右侧，导出为BRD文本并复制到剪贴板。

4. AI意图识别与更新机制

意图模型维护

· AI在内部持续维护一个结构化的意图模型（G/E/C/S四元组）。
· 每一轮对话后，AI分析人类最新输入，判断意图模型四个维度中任一维度是否发生结构性变化（新目标、方向切换、新增/修改约束、状态阶段性改变）。
· 若发生变化，则更新内部意图模型，并在回复中提示“意图模型已更新：[变化摘要]”；若仅微调细节（如细化同一方向），则不触发更新提示，但内部模型仍会调整。
· 右栏/顶部面板内容随内部意图模型同步刷新。

意图转折点

· 只有结构性变化才会触发更新提示，避免频繁打扰。
· 结构性变化的定义：
  · 目标：核心问题发生改变或显著细化
  · 探索：方向切换、假设推翻、新方法提出
  · 约束：新增或移除重要边界条件
  · 状态：阶段转换（探索→验证、卡住→突破等）

5. 人机协作闭环

1. 人在左栏输入探索意图。
2. AI回复并可能更新意图模型，在对话中提示变化。
3. 人可在右栏/顶部面板随时查看、编辑意图模型，进行校准。
4. 修改后的意图模型自动作为系统提示注入下一轮对话。
5. 人可随时导出意图模型为BRD，用于交接或存档。

6. 导出

· 将意图模型的四个字段按顺序拼接为Markdown：
  ```
  # 意图文档
  生成时间：2025-05-16 14:30
  
  ## 目标
  ...
  
  ## 当前探索
  ...
  
  ## 约束
  ...
  
  ## 状态
  ...
  ```
· 导出内容可直接作为新AI实例的上下文，也可作为人类阅读的BRD。

7. 设计原则

· 直接意图：展示人类当前认知状态，而非离散事件卡片
· 人机共维：AI主动更新，人类随时修正，共同维护意图模型
· 极简：无装饰，无额外视图切换
· 双形态统一：桌面双栏同步，移动端顶部可折叠，共享同一意图模型
