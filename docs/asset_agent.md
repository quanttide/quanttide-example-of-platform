# asset-agent — 智能体协作质量看板

## 一句话

三个视图回答三个问题：谁在做事、做得怎么样、花得值不值。

## 解决了什么业务问题

创始人面对多个 AI 智能体在资产仓库里协作，有三个说不清：

1. **说不清谁负责什么** — 智能体各管哪块、上下游是谁
2. **说不清协作效率** — 一个功能改了几轮才定下来、瓶颈出在谁身上
3. **说不清成本** — 每次调用读了多少文件、token 花在哪、有没有读不该读的

## 三个视图

**结构视图** — 打开就看到仓库目录树，每个目录标注了由哪个智能体负责。谁管 docs、谁管 src、谁管 test，一目了然。回答"谁在做事"。

**演化视图** — 按时间轴展示一个功能从设计到交付经历了几个版本：DRD 出了几版设计、前端改了几轮、后端迭代了几次、每次是谁触发的、反馈了几轮、最终在哪一版定下来的。回答"做得怎么样"。

**审计视图** — 每次智能体调用的费用清单：这次调用花了多少 token、加载了哪些文件、每个文件有没有被实际引用、有没有读自己不该读的文件。回答"花得值不值"。

## 当前建成了什么

| 模块 | 状态 | 能看到什么 |
|------|------|-----------|
| 结构视图 | ✅ 能用 | 目录树 + 智能体标签 |
| 演化视图 | ✅ 能用 | 三列版本网格 + 触发/反馈连线 |
| 审计视图 | ✅ 能用 | token 消耗 + 浪费率 + 边界检查 |
| 边界配置 | ✅ 能用 | 每个智能体的文件读取范围设置 |
| 规则引擎 | 🔧 设计中 | 文件变化自动触发智能体 |
| 统一看板 | 📋 待建 | 三个视图合成一个界面 |

## 下一步

三个视图各自已验证可用，下一步是把它们合成一个统一看板——打开一个页面，同时看到分工、效率和成本。
