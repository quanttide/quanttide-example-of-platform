# asset-agent 问题领域整合

两个仓库各自实现了 asset-agent 原型，从不同角度切入同一问题域。

## 核心问题

多智能体在共享资产仓库中协作的质量管理，分为三个层次：

| 层次 | 问题 | qtcloud-asset | qtcloud-agent |
|------|------|--------------|--------------|
| 1 分工可见 | 谁负责什么、边界在哪 | 目录树 + 角色 badge | 规则文件 + 边界编辑器 |
| 2 过程可见 | 迭代了几轮、瓶颈在哪、收敛没 | 版本演化图 | 无 |
| 3 成本可见 | token 花得值不值、越权没有 | 无 | 审计面板 + waste rate |

## 整合逻辑

三个层次串成决策链：先看分工（知职责）→ 再看过程（知效率）→ 再看成本（知资源）。缺少任何一层，管理者都无法完整判断智能体协作质量。

## 当前状态

两个原型各自验证了各自的设计点，尚未进入整合阶段。
